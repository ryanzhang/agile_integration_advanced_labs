Enrich Content and Aggregate Data on Fuse Online

In this lab, you will extend the integration created in previous lab in Fuse Online to expose it as a REST API.

Goals

  • Design Fuse Online Integration that consumes JSON records from REST API.

  • Enrich using backend REST service

  • Enrich using backend SOAP service using the REST wrapper.

  • Map the responses from backend services to data fields.

  • Save the aggregated message in an SQL database

  • Send response to the API consumer.

    Figure 1. Fuse Online Rest Enrich Integration

1. Develop the Solution

Develop the solution in the Fuse Online console. You will be provided a Red Hat Fuse Online console URL by your instructor. It should be of the format https://user$seq-fuse-online.$OCP_DOMAIN.

You should be able to login with your OCP userid and password.

1.1. Edit the Integration

We will be creating the new integration as shown in the integration flow diagram above.

  1. Click on Create a New Integration.

  2. Choose API Provider as the start connection.

  3. Choose to create from scratch.

  4. You should get an Apicurito design page now. Enter the details of below:

    1. Name: AccountUploadAPI

    2. Path: /rest/account

    3. Operation: PUT

    4. Data Type: Account

    5. Data Type Example: Copy the data below:

      {"company":{"name":"Rotobots","geo":"NA","active":true},"contact":{"firstName":"Bill","lastName":"Smith","streetAddr":"100 N Park Ave.","city":"Phoenix","state":"AZ","zip":"85017","phone":"602-555-1100"}}

    6. PUT Summary: Create Account Enrich and Upload record

    7. Consumes(inputs): application/json

    8. Produces(outputs): application/json

    9. RequestBodyType: Account

    10. Response: 200 OK

    11. Response Type: String

    12. Response Example: Copy the data below:

      {"result": "Account created successfully."}

  6. Save the changes.

  7. Name the integration to APIToSQLEnrich.

  8. Choose the PUT operation.

  9. Add the connections and steps as in the previous lab.

  10. Finally, add a Data Mapping step to respond to the API request.

1.2. Publish & Test the API

  1. Save and publish the integration.

  2. The URL to the API should be available from the integration page.

  3. You can access the REST API specification by opening the URL: <i-apitosqlenrich route>/openapi.json

  4. Send a curl request to the API.

    curl -k <i-apitosqlenrich route>/rest/account -X PUT  -d '{"company":{"name":"Rotobots","geo":"NA","active":true},"contact":{"firstName":"Bill","lastName":"Smith","streetAddr":"100 N Park Ave.","city":"Phoenix","state":"AZ","zip":"85017","phone":"602-555-1100"}}' -H 'content-type: application/json'

  5. You should see a response as below:

    {"result": "Account created successfully."}

  6. Also verify the Activity Log of the integration and the SQL table T_ACCOUNT to ensure the request is processed successfully.

Congratulations, you have completed this lab.
