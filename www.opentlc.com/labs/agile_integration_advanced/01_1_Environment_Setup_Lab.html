Agile Integration Advanced Environment Setup Lab

In this lab, you set up the environment for the labs in this course.

Goals

  • Set up the developer tools on your desktop

  • Authenticate the OpenShift client with the OpenShift master API

  • Install and use the OpenShift command line interface (CLI)

Prerequisites

  • An open broadband Internet connection

  • A terminal client, such as PuTTy for Windows

  • An up-to-date browser, such as Google Chrome or Firefox

  • OPENTLC credentials

  • Enough disk space to install Red Hat Developer Studio, Red Hat Fuse, Red Hat AMQ and other programming tools (maven, oc client, java).

    Make sure that the Internet connection is free of corporate proxies or other network rules that prevent access to remote servers on the Internet.

1. Install Required Software

In this lab, you create Fuse on OpenShift applications using Red Hat JBoss Developer Studio and OpenShift CLI tools on the desktop and deploy them to an OpenShift project.

  1. Install the software listed here on your local machine:

    • Java SE (version 1.8)

    • Apache Maven (version 3.3.9+)

    • Git (latest version)

    • OpenShift CLI client (version 3.9)

    • (Optional) SoapUI (latest version)

2. Install JBoss Developer Studio

JBoss Developer Studio is an integrated development environment (IDE) that combines both tooling and runtime components, including Eclipse plug-ins, best-of-breed open source tools, and the Red Hat® JBoss® Enterprise Application Platform (JBoss EAP).

To complete the labs in the course, you must have JBoss Developer Studio installed in your local development environment. You use JBoss Developer Studio to design Apache Camel routes.

2.1. Install JBoss Developer Studio 12.9.0

  1. Using your browser, navigate to the product page for JBoss Developer Studio.

  2. On the left side, click Download.

    • Note that the latest release of JBoss Developer Studio is highlighted near the top of the page.

  3. Click Installer to download the installer for version 12.9.0 of JBoss Developer Studio:

  4. Proceed to log in.

    • The download begins shortly after you log in.

  5. Follow the onscreen instructions to install JBoss Developer Studio, substituting the name of the JAR file with the one you downloaded:

    The installation guide is also available on the JBoss Developer Studio Product Documentation page, where you can select the version you are using.

2.2. Install JBoss Developer Studio Integration Stack Plug-ins

JBoss Developer Studio includes a variety of plug-ins for Eclipse. You use the following JBoss Developer Studio plug-ins to complete the labs in the Red Hat OPEN middleware courses:

  • Integration Stack: The Integration Stack suite of plug-ins is particularly important when using Red Hat® Fuse and Red Hat® AMQ. The Integration Stack is not included with JBoss Developer Studio and must be manually installed.

  • EGit: JBoss Developer Studio includes the Eclipse EGit plug-in, which provides Git project support. No additional installation is required. Git is an open source version control system, providing developers with fast, versatile access to their application code’s entire revision history.

  • M2E: JBoss Developer Studio includes the Eclipse M2E plug-in, which provides support for Apache Maven projects. No additional installation is required. The M2E plug-in enables you to edit a Maven project’s pom.xml and run a Maven build from the IDE.

You can select the plugin installation during the Developer Studio install process, or you can select these from the welcome page.

3. Obtain Course Lab Assets

This course comes with a variety of lab assets that are version-controlled in GitHub. In this section, you clone or update the lab assets on your desktop so that they are available locally for use in the course’s other labs.

3.1. Clone Lab Project from Git

  1. In a terminal shell, create a directory named ai_advanced_labs and navigate to it:

    $ mkdir ai_advanced_labs
$ cd ai_advanced_labs

  2. Import the lab assets from GitHub:

    $ git clone https://github.com/gpe-mw-training/agile_integration_advanced_labs.git
$ cd agile_integration_advanced_labs
    This is the labs root folder. The absolute path to this folder is referred to as $AI_EXERCISE_HOME in the instructions.

4. Install Red Hat Fuse 7.2.0 (Optional)

If you wish to deploy and run the labs locally, you will need to install Red Hat Fuse on your laptop. Please download the Red Hat Fuse install from Red Hat Fuse Download and follow the installation steps.

Note that the skeleton code for the labs is provided as springboot application, so you can choose to test the labs locally using the spring-boot maven plugin or from Red Hat Developer Studio. Optionally, you can install standalone Red Hat Fuse Karaf or EAP installer depending on your preference, and can deploy the labs to these standalone environments. As part of the labs, you will also be deploying the labs to an OpenShift environment.

5. Install Red Hat AMQ 7.2.0

You need to run a Red Hat AMQ broker locally in order to ecxecute some of the labs in this course. Hence, please download the Red Hat AMQ 7.2.0 Broker installer and follow the instructions to install on your laptop.

Once installed, please follow the link: Getting Started steps to start a new broker running locally on your laptop.

Provide the userid and password for the broker as admin and password.

6. Prepare OpenShift Project

A workshop OpenShift cluster is provisioned for use during the class. The GUID and login information will be provided by your instructor.

6.1. Create OpenShift Project

Now that you have provisioned the lab environment, you can log in to the OpenShift Container Platform web console and create a project.

  1. In a browser, navigate to the URL of the OpenShift shared cluster environment provided to you --for example, https://master.na39.openshift.opentlc.com:

    Pay close attention to the region code. In the example above, the region code is na39, but your region may be different.
    Use Google Chrome, Firefox, or Internet Explorer to access the OpenShift Container Platform web console. The Safari browser does not display the page correctly.

  2. Provide your username and password to log in to the OpenShift Container Platform web console.

  3. Create a new project for deploying your Red Hat Fuse on OpenShift applications.

  4. In the New Project dialog, enter the following:

    • Name: $OCP_PROJECT_PREFIX-fuse

    • Display Name: Fuse Experienced Project

      $OCP_PROJECT_PREFIX is a unique prefix for your project. You can set it to your initials, or the USERID. Please remember the value you have set, this needs to be used for all your projects created.

  5. Click Create.

6.2. Authenticate OpenShift Client

The OpenShift environment is ready for you to install OpenShift applications. You interact with this environment via the OpenShift CLI or the OpenShift Container Platform web console. Before you can use the OpenShift client, you must authenticate with the OpenShift master API.

  1. On your desktop, open a terminal shell using, for example, PuTTY for Windows.

  2. Authenticate with the OpenShift master API, replacing the value after the -u parameter with your Red Hat OPENTLC username, and the value after //master. with the region specified in your provisioning email.

    $ oc login https://master.$REGION.openshift.opentlc.com:443 -u $USERID

Password:
Login successful.

6.3. Deploy AMQ Broker on OpenShift

We need to deploy AMQ 7 broker on OpenShift. The general installation steps are documented here: AMQ Installation of OpenShift Guide.

  1. In the terminal, login to the OCP cluster.

  2. Choose the commands:

    $ oc project $OCP_PROJECT_PREFIX-fuse

  3. Add the view role to the service account:

    $ echo '{"kind": "ServiceAccount", "apiVersion": "v1", "metadata": {"name": "amq-service-account"}}' | oc create -f -
$ oc policy add-role-to-user view system:serviceaccount:$OCP_PROJECT_PREFIX-fuse:amq-service-account

  4. Deploy the broker:

    $ oc new-app --template=amq-broker-72-basic \
   -e AMQ_PROTOCOL=openwire,amqp,stomp,mqtt,hornetq \
   -e AMQ_USER=admin \
   -e AMQ_PASSWORD=password \
   -e AMQ_ROLE=admin

  5. Check that the broker pod is running.

  6. Make a note of the console URL. You can access the console URL from a web browser and ensure the broker is started correctly.

  7. The broker service url for AMQP would be broker-amq-amqp:5672. Make a note of this when configuring the AMQ connection for the labs.

6.4. Deploy PostgreSQL on OpenShift

We need to deploy PostgreSQL database on OpenShift. The general installation steps are documented here: PostgreSQL on OpenShift guide.

  1. In the terminal, login to the OCP cluster.

  2. Choose the commands:

    $ oc project $OCP_PROJECT_PREFIX-fuse

  3. Deploy the database:

    $ oc new-app \
    -e POSTGRESQL_USER=postgres \
    -e POSTGRESQL_PASSWORD=postgres \
    -e POSTGRESQL_DATABASE=sampledb \
    postgresql-persistent

  4. Check that the database pod is running.

  5. Identify the name of the pod running PostgreSQL and open a remote shell to the pod:

    $  oc get pods | grep postgresql
$  oc rsh <pod>

  6. Login to PostgreSQL and create the sampledb database.

    sh-4.2$ createdb -h localhost -p 5432 -U postgres sampledb
sh-4.2$ PGPASSWORD=$POSTGRESQL_PASSWORD psql -h postgresql $POSTGRESQL_DATABASE $POSTGRESQL_USER
psql (9.6.10)
Type "help" for help.

sampledb=#

  7. Create the tables. Run the following commands on the postgresql command line.

    CREATE SCHEMA USECASE;
CREATE TABLE USECASE.T_ACCOUNT (
    id  SERIAL PRIMARY KEY,
    CLIENT_ID integer,
    SALES_CONTACT VARCHAR(30),
    COMPANY_NAME VARCHAR(50),
    COMPANY_GEO CHAR(20) ,
    COMPANY_ACTIVE BOOLEAN,
    CONTACT_FIRST_NAME VARCHAR(35),
    CONTACT_LAST_NAME VARCHAR(35),
    CONTACT_ADDRESS VARCHAR(255),
    CONTACT_CITY VARCHAR(40),
    CONTACT_STATE VARCHAR(40),
    CONTACT_ZIP VARCHAR(10),
    CONTACT_EMAIL VARCHAR(60),
    CONTACT_PHONE VARCHAR(35),
    CREATION_DATE TIMESTAMP,
    CREATION_USER VARCHAR(255)
);
CREATE TABLE USECASE.T_ERROR (
    ID SERIAL PRIMARY KEY,
    ERROR_CODE VARCHAR(4) NOT NULL,
    ERROR_MESSAGE VARCHAR(255),
    MESSAGE VARCHAR(512),
    STATUS CHAR(6)
);
    You can use \q to exit the postgreSQL command line.

Congratulations, your environment is set up correctly and you can proceed with the rest of the labs.
