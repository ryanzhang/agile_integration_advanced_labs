Discover & Manage API with 3scale

In this lab, you change the Fuse Online integration API to make the service discoverable by 3scale. You will then manage the API in 3scale and ensure the requests are routed to the integration through the 3scale API Gateway.

Goals

  • Change the Integration Service

  • Discover Integration Service in 3scale

  • Create Application Plan & Application in 3scale

  • Test the API request via 3scale

1. Develop the Solution

Develop the solution by following the steps below.

1.1. Service

The i-apitosqlenrich service created as part of the Fuse Online lab needs to be edited to make the service discoverable by 3scale. This ensures that 3scale can automatically scan the services in the OpenShift cluster and ensure that the service is imported correctly.

  1. Edit the service i-apitosqlenrich in the OpenShift project user$seq-fuse.

    oc edit svc i-apitosqlenrich

  2. Add the following line under labels:

        discovery.3scale.net: "true"

  3. Add the following lines under annotations:

        discovery.3scale.net/description-path: /openapi.json
    discovery.3scale.net/path:
    discovery.3scale.net/port: "8080"
    discovery.3scale.net/scheme: http

  4. Save the changes.

1.2. 3scale API Management

The rest of the configuration needs to be done in the 3scale API Management Platform admin console. Please login to your 3scale admin portal with the userid and password details provided by the instructor.

1.2.1. Discovery

  1. In the home page, click on New API.

  2. Select the button Import from OpenShift. Select your Fuse Online namespace and corresponding i-apitosqlenrich service.

  3. Click on Create Service. The service will be imported within a few minutes and appear in the list of integrations.

1.2.2. Integration

  1. Click on Integrate this API next to the API service.

  2. Provide the Staging Public Base URL and Production Public Base URL corresponding to the OpenShift route to your apicast-staging and apicast-production apps.

  3. Add a new mapping rule:

    1. verb: PUT

    2. Pattern: /

    3. +: 1

    4. Metric or Method: hits

  4. Leave the rest of the default values unchanged and click on Update & test in Staging Environment.

  5. Click on Create Application Plan

    1. Name: basic

    2. System name: i-apitosqlenrich/basic

  6. Create the application plan.

  7. Publish the application plan.

1.2.3. Application

  1. Click on Audience.

  2. Click on 1 under the Apps for Developer account.

  3. Click on Create Application.

  4. In the Create Application page:

    1. Application Plan: basic

      This should be the same application plan you created in the integration.

    2. Name: DeveloperAccountPlan

    3. Description: Developer Account Plan

  5. Click on Create Application.

  6. Note the API Credentials user key. You need this to make a request to the API on 3scale.

1.3. Test

  1. Use the following curl command to make a request to the 3scale staging route:

    curl -k <user$seq staging route>/rest/account?user_key=<user$seq application key> -X PUT  -d '{"company":{"name":"Rotobots","geo":"NA","active":true},"contact":{"firstName":"Bill","lastName":"Smith","streetAddr":"100 N Park Ave.","city":"Phoenix","state":"AZ","zip":"85017","phone":"602-555-1100"}}' -H 'content-type: application/json'

  2. If the request is successful you should receive a response as below:

    {"result": "Account created successfully."}

  3. Also check the Analytics in 3scale to ensure the request is recorded.

  4. Verify the Fuse Online integration Activity log to monitor the request.

1.4. Bonus

You can promote the service to production in 3scale and verify that the API request to production works successfully.

Congratulations, you have completed this lab.
