Enrich Content and Aggregate Data on Fuse Online

In this lab, you develop an integration in Fuse Online that process the account records in JSON format, and through enrichment and aggregation create a more complete account record. The resulting record is stored in a database.

Goals

  • Design Fuse Online Integration that consumes JSON records from AMQ

  • Enrich using backend REST service

  • Enrich using backend SOAP service using the REST wrapper.

  • Map the responses from backend services to data fields.

  • Save the aggregated message in an SQL database

    Figure 1. Fuse Online Enrich Integration

1. Develop the Solution

Develop the solution by creating a new integration in the Fuse Online console. You will be provided a Red Hat Fuse Online console URL by your instructor. It should be of the format https://user$seq-fuse-online.$OCP_DOMAIN.

IMP: user$seq represents the username in the OCP cluster. Replace with your assigned username, e.g user1, user2, etc.

You should be able to login with your OCP userid and password.

1.1. Extensions

A custom extension is required in order to manage the headers for the request to backend APIs. Hence, add the following extension: https://github.com/syndesisio/syndesis-extensions/tree/master/syndesis-extension-manage-headers

The extension is available in the lab assets location $AI_EXERCISE_HOME/code/fuse_online/extensions/syndesis-extension-manage-headers-1.0.0.jar

1.2. API Client Connectors

Create API connectors for the backend API services.

  1. Upload the REST API spec for the REST CXFRS Service swagger spec using the URL: http://rest-cxfrs-service-user$seq-fuse-online.$OCP_DOMAIN/rest/swagger.json

    • Provide the following details:

      1. Connector Name: Rest CXFRS Service

      2. Host: http://cxfrs-rest-service.user$seq:8080

      3. Base URL: /rest

  2. Upload the SOAP API spec for the Legacy SOAP REST Wrapper API JSON from the lab asset location: fuse/05_migrate_fuse6/legacy-soap-rest-wrapper/src/main/resources/swagger.json

    • Provide the following details:

      1. Connector Name: SOAP CXFWS Service

      2. Host: http://legacy-soap-rest-wrapper.user$seq:8080

      3. Base URL: /api

1.3. Connection

Set up the connections to following endpoints used for the integration:

1.3.1. AMQP Message Broker : Use the AMQ broker deployed in the user$seq-fuse project with following details:

  1. Connection URI: amqp://broker-amq-amqp.user$seq-fuse:5672

  2. User Name: admin

  3. Password: password

  4. Retain rest of the values as default and validate the connection.

  5. Name: AMQPBrokerConnection

1.3.2. Database: Use the PostgreSQL database deployed in the user$seq-fuse project with following details:

  1. Connection URL: jdbc:postgresql://postgresql.user$seq-fuse:5432/sampledb

  2. Username: postgres

  3. Password: postgres

  4. Schema: USECASE

  5. Name: UsecaseDB

    There is an intermittent issue in Fuse Online which means the integration does not recognize the schema. If you run into this, move the T_ACCOUNT & T_ERROR tables to the public schema, and the integration should work.

1.3.3. Rest CXFRS Service: set up the connection to the API client connector.

  1. Authentication Type: No security

  2. Host: http://rest-cxfrs-service.user$seq-fuse:8080

  3. Base path: /rest

  4. Name: RESTEnrichAPI

1.3.4. SOAP CXFWS Service: set up the connection to the API client connector.

  1. Authentication Type: No security

  2. Host: http://legacy-soap-rest-wrapper.user1-fuse:8080

  3. Base path: /api

  4. Name: SOAPEnrichAPI

1.4. Develop the Integration Module

Develop the Integration as per the above route. Use the following steps:

  1. Name the Integration AMQToSQLEnrich

  2. Start Integration from AMQPBrokerConnection:

    1. Subscribe for messages

    2. Destination: accountQueue

    3. Destination Type: Queue

  3. Select Output type:

    1. Select Type: JSON Instance

    2. Definition: Paste the following:

      {"company":{"name":"Rotobots","geo":"NA","active":true},"contact":{"firstName":"Bill","lastName":"Smith","streetAddr":"100 N Park Ave.","city":"Phoenix","state":"AZ","zip":"85017","phone":"602-555-1100"}}

    3. Data Type Name: Account

  4. Choose Finish Connection UsecaseDB

    1. Action: Invoke SQL

    2. SQL Statement: Paste the following:

      INSERT INTO T_ACCOUNT(CLIENT_ID,SALES_CONTACT,COMPANY_NAME,COMPANY_GEO,COMPANY_ACTIVE,CONTACT_FIRST_NAME,CONTACT_LAST_NAME,CONTACT_ADDRESS,CONTACT_CITY,CONTACT_STATE,CONTACT_ZIP,CONTACT_PHONE,CREATION_DATE,CREATION_USER)                          VALUES                          (:#CLIENT_ID,:#SALES_CONTACT,:#COMPANY_NAME,:#COMPANY_GEO,:#COMPANY_ACTIVE,:#CONTACT_FIRST_NAME,:#CONTACT_LAST_NAME,:#CONTACT_ADDRESS,:#CONTACT_CITY,:#CONTACT_STATE,:#CONTACT_ZIP,:#CONTACT_PHONE,current_timestamp,:#CREATION_USER);

  5. Add Step Log

    1. Check Message Body

    2. Custom Text: Input Message

  6. Add Connection RestEnrichService

    1. Choose Action POST /customerservice/enrich

  7. Add Data Mapping before the RestEnrichService connection.

    1. Map corresponding fields in input with the expected JSON request.

  8. Add Connection SoapEnrichService

    1. Choose Action Enrich and Update a user account.

  9. Add Data Mapping before the SoapEnrichService connection.

    1. Map corresponding fields in input with the expected JSON request.

  10. Add a Step Log immediately after the RestEnrichService.

    1. Check Message Body

    2. Custom Text: After REST Service

  11. Add a Step Remove Header immediately after the RestEnrichService.

    1. Enter Header Transfer-Encoding.

      The Transfer-Encoding header is addded by camel during the call to REST service. This header will cause errors if passed to the subsequent SOAP service. Hence, the header needs to be removed before the next API call.

  12. Add a Step Log immediately after the SoapEnrichService.

    1. Check Message Body

    2. Custom Text: After SOAP Service

  13. Add a Data Mapping step before the database connection.

    1. Map the corresponding fields.

    2. Enter a constant value fuse_online and map it to CREATION_USER.

2. Useful SQL Scripts

CREATE TABLE T_ACCOUNT ( id SERIAL PRIMARY KEY, CLIENT_ID integer, SALES_CONTACT VARCHAR(30), COMPANY_NAME VARCHAR(50), COMPANY_GEO CHAR(20) , COMPANY_ACTIVE BOOLEAN, CONTACT_FIRST_NAME VARCHAR(35), CONTACT_LAST_NAME VARCHAR(35), CONTACT_ADDRESS VARCHAR(255), CONTACT_CITY VARCHAR(40), CONTACT_STATE VARCHAR(40), CONTACT_ZIP VARCHAR(10), CONTACT_EMAIL VARCHAR(60), CONTACT_PHONE VARCHAR(35), CREATION_DATE TIMESTAMP, CREATION_USER VARCHAR(255) ); CREATE TABLE T_ERROR ( ID SERIAL PRIMARY KEY, ERROR_CODE VARCHAR(4) NOT NULL, ERROR_MESSAGE VARCHAR(255), MESSAGE VARCHAR(512), STATUS CHAR(6) );
